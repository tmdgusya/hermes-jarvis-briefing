# Jarvis Threads 데모 진행 가이드

이 문서는 촬영/시연할 때 Jarvis 플러그인을 어떻게 켜고, 어떤 순서로 말하면 되는지 정리한 데모용 runbook입니다.

## 데모 목표

박수 2번으로 Jarvis가 켜지고 화면에 `JARVIS ON`이 뜬 뒤, 사용자가 자연어로 Threads 인기글 브리핑을 요청하고 3~5턴 정도 이어서 대화하는 장면을 보여줍니다. 대화 중에는 웹 오버레이의 원형 파형이 실제 마이크 입력에 반응해서 일렁입니다.

데모는 실제 Threads API나 웹 크롤링을 기다리지 않습니다. 촬영 중 지연/실패/결과 변동을 막기 위해 `/jarvis demo`는 스크립트 기반으로 동작합니다.

## 준비 체크리스트

1. Hermes Agent가 설치되어 있어야 합니다.
2. 이 플러그인이 설치되어 있어야 합니다.

   ```bash
   hermes plugins install tmdgusya/hermes-jarvis-briefing
   ```

   개발용 로컬 clone을 쓰는 경우:

   ```bash
   git clone https://github.com/tmdgusya/hermes-jarvis-briefing ~/.hermes/plugins/jarvis-briefing
   ```

3. 마이크 권한이 터미널 앱에 허용되어 있어야 합니다.
4. 음성 기능이 준비되어 있어야 합니다.

   ```bash
   pip install 'hermes-agent[voice]'
   ```

5. 촬영 전에 조용한 환경에서 박수 2번이 잘 감지되는지 1회 리허설하세요.

## 추천 촬영 세팅

- 화면에는 Hermes CLI와 Jarvis 웹 오버레이가 모두 보이게 둡니다.
- 웹 오버레이가 주인공이면 CLI는 작게 두고 오버레이를 크게 보여주세요.
- 브라우저가 마이크 권한을 요청하면 허용하세요. 이 권한은 음성 인식용이 아니라 웹 오버레이의 물결/파형을 실제 목소리에 맞춰 움직이기 위한 것입니다.
- 모델은 빠른 모델을 쓰는 것을 추천합니다. 데모 응답 자체는 스크립트 기반이라 모델이 조금 달라도 흔들림이 적습니다.
- 마이크 입력이 너무 작으면 박수 감지가 안 될 수 있으니, 노트북 내장 마이크 기준으로 화면 가까이에서 또렷하게 박수 2번을 치세요.

## 전체 데모 순서

### 1. Hermes 실행

터미널에서 실행합니다.

```bash
hermes chat
```

### 2. 데모 모드 시작

Hermes 세션 안에서 아래 명령을 입력합니다.

```text
/jarvis demo
```

또는 같은 동작의 별칭을 사용할 수 있습니다.

```text
/jarvis threads
```

### 3. 준비 비프 후 박수 2번

콘솔에 Jarvis가 listening 상태라고 표시되고 짧은 준비 비프가 들립니다.

그 다음 3초 안에 박수를 2번 칩니다.

```text
👏👏
```

정상 동작하면 확인 비프가 들리고 웹 오버레이가 아래 상태로 바뀝니다.

```text
JARVIS ON
```

이때 Jarvis가 자동으로 음성 녹음을 시작합니다. 터미널에 `● Recording...` 이 보이면 바로 첫 질문을 말하세요. 별도로 Ctrl+B를 누를 필요가 없습니다.

### 4. 첫 번째 대사

`JARVIS ON`이 뜬 뒤 아래 문장을 그대로 말합니다.

```text
스레드에서 72시간 동안 #바이브코딩 키워드로 발행된 인기글 브리핑해줘
```

### 5. 기대 응답

Jarvis는 아래 흐름으로 답합니다.

```text
스레드 수집 중... 완료.
지난 72시간 #바이브코딩 인기글 Top 3입니다.
1위는 "기획자가 혼자 앱 만든 후기", 좋아요 2.4천.
2위는 "개발자 없이 창업한 이야기", 좋아요 1.8천.
3위는 "바이브코딩 3개월 수익 공개", 좋아요 1.2천입니다.
```

응답이 끝나면 continuous voice mode로 다시 `● Recording...` 상태가 열립니다.

### 6. 이어서 대화하기

Jarvis 응답이 끝날 때마다 바로 이어서 한 문장씩 물어보면 됩니다.

```text
공통점이 있을까?
```

```text
그러면 내가 오늘 올릴 만한 건 뭐가 좋을까?
```

```text
하나만 골라줘.
```

```text
그걸 캡션까지 써줘.
```

### 7. 기대 후속 응답

Jarvis는 Top 3의 공통점을 요약한 뒤, 콘텐츠 아이디어와 최종 캡션까지 이어서 제안합니다. 핵심 포인트는 다음과 같습니다.

- 비개발자/1인 창작자가 직접 제품을 만든 이야기
- 개발 과정보다 결과와 수익/성과가 강조됨
- “나도 해볼 수 있다”는 바이브코딩 서사가 강함
- 후기, 수익 공개, 창업 스토리처럼 경험담 형식이 반응이 좋음

## 촬영용 대본

아래 순서대로 진행하면 됩니다.

```text
[액션]

[나]
스레드에서 72시간 동안 #바이브코딩 키워드로 발행된 인기글 브리핑해줘

[자비스]
스레드 수집 중... 완료. 지난 72시간 #바이브코딩 인기글 Top 3입니다.
1위는 "기획자가 혼자 앱 만든 후기", 좋아요 2.4천.
2위는 "개발자 없이 창업한 이야기", 좋아요 1.8천.
3위는 "바이브코딩 3개월 수익 공개", 좋아요 1.2천입니다.

[나]
공통점이 있을까?

[자비스]
공통점은 세 가지입니다.
첫째, 모두 비개발자나 1인 창작자가 AI로 직접 결과물을 만들었다는 서사가 있습니다.
둘째, 후기·창업·수익 공개처럼 결과가 숫자나 경험으로 검증됩니다.
셋째, 제목이 "나도 해볼 수 있겠다"는 대리 가능성을 강하게 자극합니다.

[나]
그러면 내가 오늘 올릴 만한 건 뭐가 좋을까?

[자비스]
오늘 올릴 거라면 "제가 개발자 없이 3시간 만에 만든 자동화" 같은 체험형 포맷이 좋습니다.
구조는 문제 하나, 만든 과정 한 장면, 결과 숫자 하나로 가면 됩니다.
훅은 "코딩을 몰라도 이 정도는 만들 수 있더라"가 가장 안전합니다.

[나]
하나만 골라줘.

[자비스]
하나만 고르면 "개발자 없이 만든 작은 자동화가 실제 시간을 얼마나 줄였는지"로 가겠습니다.
숫자가 들어가서 신뢰가 생기고, 비개발자도 따라 할 수 있다는 바이브코딩 맥락과 잘 맞습니다.

[나]
그걸 캡션까지 써줘.

[자비스]
캡션 초안입니다. "개발자는 아니지만, 오늘 반복 업무 하나를 AI로 자동화해봤습니다.
걸린 시간은 3시간, 줄어든 시간은 매주 4시간.
바이브코딩의 장점은 완벽한 코드를 쓰는 게 아니라, 내 문제를 직접 해결하는 속도에 있는 것 같습니다."
```

## 실패했을 때

### 박수 감지가 안 될 때

- 터미널/IDE/앱에 마이크 권한이 있는지 확인하세요.
- 박수를 더 크게, 0.3초~3초 간격 안에서 두 번 치세요.
- 타임아웃 메시지에 표시되는 peak RMS와 제안값을 보고 `clap_detector.py`의 임계값을 낮추면 됩니다.

### 웹 오버레이가 안 뜰 때

- Hermes 세션을 종료 후 다시 실행하세요.
- 이전에 열린 브라우저/웹뷰가 있으면 닫고 다시 `/jarvis demo`를 실행하세요.

### 웹 오버레이 물결이 안 움직일 때

- 브라우저 주소가 `http://127.0.0.1:8765/...` 인지 확인하세요. localhost/127.0.0.1에서는 마이크 권한을 요청할 수 있습니다.
- 브라우저가 마이크 권한을 물으면 허용하세요. 거부하면 기본 애니메이션은 보이지만 실제 목소리 크기에는 반응하지 않습니다.
- macOS에서 브라우저 마이크 권한도 확인하세요: 시스템 설정 → 개인정보 보호 및 보안 → 마이크 → 사용 중인 브라우저 허용.
- 화면 녹화 시에는 브라우저 오버레이를 크게 띄우고 말할 때 원형 파형이 커지는지 확인하세요.

### Jarvis가 바로 답하지 않을 때

- `/jarvis demo` 직후에는 의도적으로 바로 답하지 않습니다.
- `JARVIS ON` 상태가 뜬 다음 자동으로 음성 녹음이 시작되어야 합니다.
- 터미널에 `● Recording...` 이 보이면 첫 질문을 말하세요.
- 음성 입력이 끝난 뒤 3초 정도 침묵하면 자동 전송됩니다.
- 만약 `JARVIS ON`은 떴는데 `● Recording...`이 안 보이면 플러그인이 오래된 버전일 수 있습니다. `/Users/roach/.hermes/plugins/jarvis-briefing` 에서 `git pull` 후 Hermes 세션을 새로 여세요.

### 실제 Threads 데이터가 안 나오냐는 질문을 받으면

아래처럼 설명하면 됩니다.

```text
이 데모는 라이브 촬영 안정성을 위해 스크립트 기반으로 구성했습니다.
실제 수집 API나 웹 자동화를 붙이면 같은 인터페이스에서 데이터 소스만 교체할 수 있습니다.
```

## 종료 방법

대화/음성 모드를 끝내려면 Hermes 세션에서 `Ctrl+B`를 누르거나 세션을 종료합니다.

```text
/quit
```
