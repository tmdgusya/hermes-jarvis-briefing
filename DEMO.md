# Jarvis Threads 데모 진행 가이드

이 문서는 촬영/시연할 때 Jarvis 플러그인을 어떻게 켜고, 어떤 순서로 말하면 되는지 정리한 데모용 runbook입니다.

## 데모 목표

박수 2번으로 Jarvis가 켜지고 화면에 `JARVIS ON`이 뜬 뒤, 사용자가 자연어로 Threads 인기글 브리핑을 요청하면 Jarvis가 빠르게 Top 3와 공통점을 답하는 장면을 보여줍니다.

데모는 실제 Threads API나 웹 크롤링을 기다리지 않습니다. 촬영 중 지연/실패/결과 변동을 막기 위해 `/jarvis demo`는 스크립트 기반으로 동작합니다.

## 준비 체크리스트

1. Hermes Agent가 설치되어 있어야 합니다.
2. 이 플러그인이 설치되어 있어야 합니다.

   ```bash
   hermes plugins install tmdgusya/hermes-jarvis-briefing
   ```

   개발용 로컬 clone을 쓰는 경우:

   ```bash
   git clone https://github.com/tmdgusya/hermes-jarvis-briefing ~/.hermes/plugins/jarvis-briefing
   ```

3. 마이크 권한이 터미널 앱에 허용되어 있어야 합니다.
4. 음성 기능이 준비되어 있어야 합니다.

   ```bash
   pip install 'hermes-agent[voice]'
   ```

5. 촬영 전에 조용한 환경에서 박수 2번이 잘 감지되는지 1회 리허설하세요.

## 추천 촬영 세팅

- 화면에는 Hermes CLI와 Jarvis 웹 오버레이가 모두 보이게 둡니다.
- 웹 오버레이가 주인공이면 CLI는 작게 두고 오버레이를 크게 보여주세요.
- 모델은 빠른 모델을 쓰는 것을 추천합니다. 데모 응답 자체는 스크립트 기반이라 모델이 조금 달라도 흔들림이 적습니다.
- 마이크 입력이 너무 작으면 박수 감지가 안 될 수 있으니, 노트북 내장 마이크 기준으로 화면 가까이에서 또렷하게 박수 2번을 치세요.

## 전체 데모 순서

### 1. Hermes 실행

터미널에서 실행합니다.

```bash
hermes chat
```

### 2. 데모 모드 시작

Hermes 세션 안에서 아래 명령을 입력합니다.

```text
/jarvis demo
```

또는 같은 동작의 별칭을 사용할 수 있습니다.

```text
/jarvis threads
```

### 3. 준비 비프 후 박수 2번

콘솔에 Jarvis가 listening 상태라고 표시되고 짧은 준비 비프가 들립니다.

그 다음 3초 안에 박수를 2번 칩니다.

```text
👏👏
```

정상 동작하면 확인 비프가 들리고 웹 오버레이가 아래 상태로 바뀝니다.

```text
JARVIS ON
```

이때 Jarvis는 아직 브리핑을 시작하지 않고, 사용자의 다음 음성 명령을 기다립니다.

### 4. 첫 번째 대사

`JARVIS ON`이 뜬 뒤 아래 문장을 그대로 말합니다.

```text
스레드에서 72시간 동안 #바이브코딩 키워드로 발행된 인기글 브리핑해줘
```

### 5. 기대 응답

Jarvis는 아래 흐름으로 답합니다.

```text
스레드 수집 중... 완료.
오늘 인기글 top 3입니다.
🔥 1위 — "기획자가 혼자 앱 만든 후기" (좋아요 2.4천)
🔥 2위 — "개발자 없이 창업한 이야기" (좋아요 1.8천)
🔥 3위 — "바이브코딩 3개월 수익 공개" (좋아요 1.2천)
```

### 6. 후속 질문

Jarvis 응답이 끝나면 바로 이어서 아래처럼 물어봅니다.

```text
공통점이 있을까?
```

### 7. 기대 후속 응답

Jarvis는 Top 3의 공통점을 요약합니다. 핵심 포인트는 다음과 같습니다.

- 비개발자/1인 창작자가 직접 제품을 만든 이야기
- 개발 과정보다 결과와 수익/성과가 강조됨
- “나도 해볼 수 있다”는 바이브코딩 서사가 강함
- 후기, 수익 공개, 창업 스토리처럼 경험담 형식이 반응이 좋음

## 촬영용 대본

아래 순서대로 진행하면 됩니다.

```text
[액션]
박수 2번 — 화면이 켜지며 JARVIS ON 자막

[나]
스레드에서 72시간 동안 #바이브코딩 키워드로 발행된 인기글 브리핑해줘

[자비스]
스레드 수집 중... 완료.
오늘 인기글 top 3입니다.
🔥 1위 — "기획자가 혼자 앱 만든 후기" (좋아요 2.4천)
🔥 2위 — "개발자 없이 창업한 이야기" (좋아요 1.8천)
🔥 3위 — "바이브코딩 3개월 수익 공개" (좋아요 1.2천)

[나]
공통점이 있을까?

[자비스]
공통점은 비개발자나 1인 창작자가 AI로 직접 결과물을 만들었다는 점입니다.
특히 개발 과정 자체보다 ‘혼자 만들었다’, ‘수익이 났다’, ‘창업까지 이어졌다’는 성과 중심 스토리가 반응을 얻고 있습니다.
즉, #바이브코딩 인기글은 기술 팁보다 “나도 만들 수 있다”는 가능성을 보여주는 경험담일수록 강합니다.
```

## 실패했을 때

### 박수 감지가 안 될 때

- 터미널/IDE/앱에 마이크 권한이 있는지 확인하세요.
- 박수를 더 크게, 0.3초~3초 간격 안에서 두 번 치세요.
- 타임아웃 메시지에 표시되는 peak RMS와 제안값을 보고 `clap_detector.py`의 임계값을 낮추면 됩니다.

### 웹 오버레이가 안 뜰 때

- Hermes 세션을 종료 후 다시 실행하세요.
- 이전에 열린 브라우저/웹뷰가 있으면 닫고 다시 `/jarvis demo`를 실행하세요.

### Jarvis가 바로 답하지 않을 때

- `/jarvis demo` 직후에는 의도적으로 바로 답하지 않습니다.
- `JARVIS ON` 상태가 뜬 다음, 사용자가 첫 질문을 말해야 응답합니다.
- 음성 입력이 끝난 뒤 3초 정도 침묵하면 자동 전송됩니다.

### 실제 Threads 데이터가 안 나오냐는 질문을 받으면

아래처럼 설명하면 됩니다.

```text
이 데모는 라이브 촬영 안정성을 위해 스크립트 기반으로 구성했습니다.
실제 수집 API나 웹 자동화를 붙이면 같은 인터페이스에서 데이터 소스만 교체할 수 있습니다.
```

## 종료 방법

대화/음성 모드를 끝내려면 Hermes 세션에서 `Ctrl+B`를 누르거나 세션을 종료합니다.

```text
/quit
```
